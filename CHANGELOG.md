## [0.10.1] – 2026-05-14

### Fixed
- Mini App на десктопе: убран мёртвый ~64px отступ сверху между нативной шапкой клиента Telegram и заголовком «Список покупок». На десктопе теперь 20px (минимальный воздух), на мобилках сохранено 64px — там это пространство занимают оверлей-контролы fullscreen-режима.

### Added
- Кнопка «Убрать купленное» в прогрессе активного списка (видна, когда что-то отмечено, но список не полностью куплен). По нажатию купленные товары переезжают в новый архивный список, активный остаётся только с непокупленными. История покупок сохраняется в архиве, симметрично авто-архивации полного списка.
- Новый endpoint `POST /api/lists/{id}/archive-purchased` → `{archived_list_id, moved}`. 404 если активный список не найден или нет купленных позиций.

### Changed
- Mini App: убрана отдельная шапка с эмодзи-корзиной и подзаголовком «mini app». Активный экран начинается с заголовка «Список покупок» 28pt + пилюли «Архив», без eyebrow и даты в title. Закрытие приложения — через нативные контролы Telegram.
- Mini App: footer-CTA «Откройте чат, чтобы добавить» переработан в primary-кнопку «Добавить товары» со sparkle-иконкой. Дизейблится во время активного ingest (распознавание/парсинг).
- Mini App: primary-кнопки в светлой теме — tinted blue (`rgba(0,122,255,0.14)` + синий текст); в тёмной — solid black. Унифицировано через хук `usePrimary()`. Затрагивает Starter, EmptyState, Archive, ArchiveDetail и ChatHint.
- Mini App: архивный список и его детали показывают **время создания** списка (дата + ЧЧ:ММ) вместо даты архивации. Заголовок детали — uppercase eyebrow с датой+временем + одна строка «Список покупок» (без даты в title).
- Mini App: в архивном списке элементы больше не зачёркиваются — отображаются серым (`text2`) без strikethrough. Reuse-кнопка стала однострочной, без sub-подписи.
- Mini App: строка `ItemRow` использует `currentColor` для зачёркивания (1.5px, skip-ink off); qty при `done=true` теперь 17pt opacity 1, при `done=false` — 15pt opacity 0.7.

### Changed
- Внутри блока отмеченных товаров порядок изменён с «по позиции вставки» на «по времени отметки» — только что отмеченный встаёт наверх блока отмеченных. Если пользователь случайно поставил галочку, её можно откатить, не ища товар среди ранее отмеченных. Реализовано в `ORDER BY done ASC, checked_at DESC, position ASC` (SQL) и в оптимистичном sort'е Mini App. Колонка `position` не меняется — при снятии галочки товар возвращается в свой исходный спот среди неотмеченных.

### Added
- Mini App на мобилках (iOS/Android) запрашивает fullscreen-режим (Bot API 8.0) и блокирует swipe-to-collapse через `disableVerticalSwipes` (Bot API 7.7). Список больше не сворачивается при случайном вертикальном скролле. На старых клиентах Telegram (<7.7) поведение не меняется, на desktop/web — тоже. Контент учитывает safe-area-insets под notch / dynamic island / home-indicator.
- Привязка бота к одному топику форум-группы через `/pin`: команда сохраняет `pinned_thread_id` в новой таблице `app_settings`, `AuthMiddleware` молча игнорирует сообщения в остальных топиках. До первого `/pin` бот, как и раньше, реагирует во всех топиках (back-compat). Перепривязка — повторным `/pin` в нужном топике; сама команда `/pin` пропускается через фильтр всегда, чтобы можно было переключаться.
- Новая env-переменная `WHISPER_PROMPT` — биас-словарь для голосового распознавания. По умолчанию заполнен типичными словами из списков покупок (семена чиа, киноа, фольга, пергамент, изолента, подгузники и т.п.), но можно расширять под свою лексику.
- Одноразовый скрипт миграции `scripts/normalize_existing_items.py` — приводит уже сохранённые товары к новому формату (lowercase + бренды с канонической капитализацией). Поддерживает `--dry-run`. Идемпотентен.

### Changed
- Авто-форматирование названий товаров: все имена сохраняются строчными буквами, а токены брендов (`Coca-Cola`, `iPhone`, `Простоквашино`, `Nestlé`, `Lay's` и т.п.) сохраняют свою каноническую капитализацию. Бренды извлекает LLM в отдельное поле JSON-схемы; финальное форматирование делает детерминированный пост-процессор `bot/services/name_format.py`. Применяется к новым добавлениям из текста, голоса и фото; для накопленных записей — миграция скриптом.
- Парсер сообщений теперь распознаёт практически любой товар, который продаётся в супермаркетах, сервисах доставки и онлайн-магазинах для дома и семьи — а не только продукты питания. Поддерживаются бытовая химия и кухонные расходники (фольга, пергамент, салфетки, мешки для мусора), гигиена и косметика, бытовая техника и электроника, строительные/ремонтные товары, корм для животных, детские товары, канцелярия и др.
- Голосовое распознавание переведено с `whisper-1` на `gpt-4o-mini-transcribe` (точнее на узкой лексике). В запрос Whisper теперь передаётся `WHISPER_PROMPT` — частотные ошибки вроде «семена чиа → семена чая» снимаются на уровне STT.
- В системный промпт парсера добавлена страховка от очевидных STT-ошибок: если в тексте бессмысленное в контексте покупок словосочетание (например, «семена чая», «куркуба»), оно нормализуется в реальное название товара.

### Changed
- Купленные товары автоматически уезжают вниз списка, неотмеченные остаются сверху. Сортировка применяется в Mini App мгновенно при клике и в API-ответе через `ORDER BY done ASC, position ASC`. Колонка `position` не меняется, поэтому при снятии галочки товар возвращается в свой исходный спот среди неотмеченных.

## [0.5.0] – 2026-05-11

### Added
- Парсер достраивает единицу измерения по контексту продукта, когда пользователь указал только число: «молоко 1» → `1 л`, «сахар 500» → `500 г`, «яблоки 5» → `5 шт`. Распространяется и на текст (`parse_text`), и на фото (`parse_image`).
- Mini App обновлён до дизайна v2: свайп влево по строке открывает действия «Изменить» / «Удалить», bottom-sheet редактирует название и количество товара, iOS-стиль alert подтверждает удаление товара или архивного списка.
- Архив открывается карточкой архивного списка с действиями «Добавить в текущий список» / «Создать новый список» (все товары снимаются с галочки) и «Удалить список».
- Status banner над футером показывает прогресс разбора сообщений из чата: распознавание голосового, обработка фото, парсинг текста и итог «Добавлено N товаров». Состояние пишется ботом в новую таблицу `ingest_events` и поднимается в `/api/state`.
- Новые endpoint'ы Mini App API: `PATCH /api/items/{id}`, `DELETE /api/items/{id}`, `GET /api/archive/{id}`, `POST /api/archive/{id}/reuse`, `DELETE /api/archive/{id}`.
- Команда `/pin` (в группе) — бот шлёт приветственное сообщение с inline-кнопкой Mini App и пинит его. Используется, когда бот уже в группе и `my_chat_member`-автопин не сработал (например, после деплоя без re-join).

### Changed
- Inline-кнопка Mini App переименована «🛒 Открыть список» → «🛒 Список», чтобы совпадать с подписью menu-button.
- Polling Mini App ускорен с 3 c до 2 c, чтобы статус ингеста обновлялся почти мгновенно.
- README: добавлен шаг **Mini App domain** с `/setdomain` в разделе «Запуск» и пояснение, что без зарегистрированного домена Telegram не откроет Mini App. В блоке про `TARGET_CHAT_ID` добавлено объяснение обязательного префикса `-100<group_id>` для супергрупп.

### Fixed
- `/pin` в forum-style супергруппе теперь публикует и пинит сообщение в том же топике, из которого вызвана команда, а не в General. Прокидывается `message_thread_id` через `post_and_pin_welcome`.
- В Mini App больше не появляется литерал `null` рядом с товаром, когда количество не было указано. Источник — строка `"null"` от GPT вместо JSON null; новый нормализатор `_normalize_qty` ловит `null`/`None`/`-`/`—` и пустые строки.

## [0.2.0] – 2026-05-10

### Added
- Текстовый обработчик отправляет промежуточное сообщение «📝 Разбираю…» во время парсинга, по аналогии с фото/голосом.
- Все статусные сообщения (текст / фото / голос) привязаны reply'ем к исходному сообщению пользователя — в групповом чате видно, на какую реплику отвечает бот.
- Поддержка direct-link Mini App для групповых чатов: новая env-переменная `WEBAPP_SHORT_NAME`, в группе под ответами появляется URL-кнопка вида `https://t.me/<bot>/<short_name>`. В приватных чатах поведение прежнее (`web_app=` инлайн-кнопка).
- Автоматический приветственный пост с кнопкой Mini App при добавлении бота в групповой чат (`my_chat_member`); сообщение пинится, если у бота есть `can_pin_messages`.

### Fixed
- В групповом чате финальное подтверждение «✓ Добавил N товаров» теперь действительно появляется в чате. До этого `sendMessage`/`editMessageText` падал с ошибкой Bot API: inline-кнопка `web_app=` запрещена вне приватных чатов, исключение всплывало после записи в БД, и в итоге товары добавлялись, а сообщения от бота пользователь не видел. Клавиатура теперь подбирается по типу чата.

## [0.1.0] – 2026-05-10

### Added
- Bootstrap бота «Список покупок»: text/voice/photo ингест через OpenAI (Whisper + gpt-4o), aiogram-бот, FastAPI mini-app, SQLite-хранилище, whitelist-доступ, общий активный список с авто-архивированием при отметке всех товаров.
