## [Unreleased]

### Added
- Привязка бота к одному топику форум-группы через `/pin`: команда сохраняет `pinned_thread_id` в новой таблице `app_settings`, `AuthMiddleware` молча игнорирует сообщения в остальных топиках. ## [0.5.0] – 2026-05-11

### Added
- Парсер достраивает единицу измерения по контексту продукта, когда пользователь указал только число: «молоко 1» → `1 л`, «сахар 500» → `500 г`, «яблоки 5» → `5 шт`. Распространяется и на текст (`parse_text`), и на фото (`parse_image`).
- Mini App обновлён до дизайна v2: свайп влево по строке открывает действия «Изменить» / «Удалить», bottom-sheet редактирует название и количество товара, iOS-стиль alert подтверждает удаление товара или архивного списка.
- Архив открывается карточкой архивного списка с действиями «Добавить в текущий список» / «Создать новый список» (все товары снимаются с галочки) и «Удалить список».
- Status banner над футером показывает прогресс разбора сообщений из чата: распознавание голосового, обработка фото, парсинг текста и итог «Добавлено N товаров». Состояние пишется ботом в новую таблицу `ingest_events` и поднимается в `/api/state`.
- Новые endpoint'ы Mini App API: `PATCH /api/items/{id}`, `DELETE /api/items/{id}`, `GET /api/archive/{id}`, `POST /api/archive/{id}/reuse`, `DELETE /api/archive/{id}`.
- Команда `/pin` (в группе) — бот шлёт приветственное сообщение с inline-кнопкой Mini App и пинит его. Используется, когда бот уже в группе и `my_chat_member`-автопин не сработал (например, после деплоя без re-join).

### Changed
- Inline-кнопка Mini App переименована «🛒 Открыть список» → «🛒 Список», чтобы совпадать с подписью menu-button.
- Polling Mini App ускорен с 3 c до 2 c, чтобы статус ингеста обновлялся почти мгновенно.
- README: добавлен шаг **Mini App domain** с `/setdomain` в разделе «Запуск» и пояснение, что без зарегистрированного домена Telegram не откроет Mini App. В блоке про `TARGET_CHAT_ID` добавлено объяснение обязательного префикса `-100<group_id>` для супергрупп.

### Fixed
- `/pin` в forum-style супергруппе теперь публикует и пинит сообщение в том же топике, из которого вызвана команда, а не в General. Прокидывается `message_thread_id` через `post_and_pin_welcome`.
- В Mini App больше не появляется литерал `null` рядом с товаром, когда количество не было указано. Источник — строка `"null"` от GPT вместо JSON null; новый нормализатор `_normalize_qty` ловит `null`/`None`/`-`/`—` и пустые строки.

## [0.2.0] – 2026-05-10

### Added
- Текстовый обработчик отправляет промежуточное сообщение «📝 Разбираю…» во время парсинга, по аналогии с фото/голосом.
- Все статусные сообщения (текст / фото / голос) привязаны reply'ем к исходному сообщению пользователя — в групповом чате видно, на какую реплику отвечает бот.
- Поддержка direct-link Mini App для групповых чатов: новая env-переменная `WEBAPP_SHORT_NAME`, в группе под ответами появляется URL-кнопка вида `https://t.me/<bot>/<short_name>`. В приватных чатах поведение прежнее (`web_app=` инлайн-кнопка).
- Автоматический приветственный пост с кнопкой Mini App при добавлении бота в групповой чат (`my_chat_member`); сообщение пинится, если у бота есть `can_pin_messages`.

### Fixed
- В групповом чате финальное подтверждение «✓ Добавил N товаров» теперь действительно появляется в чате. До этого `sendMessage`/`editMessageText` падал с ошибкой Bot API: inline-кнопка `web_app=` запрещена вне приватных чатов, исключение всплывало после записи в БД, и в итоге товары добавлялись, а сообщения от бота пользователь не видел. Клавиатура теперь подбирается по типу чата.

## [0.1.0] – 2026-05-10

### Added
- Bootstrap бота «Список покупок»: text/voice/photo ингест через OpenAI (Whisper + gpt-4o), aiogram-бот, FastAPI mini-app, SQLite-хранилище, whitelist-доступ, общий активный список с авто-архивированием при отметке всех товаров.
